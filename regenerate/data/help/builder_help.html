Using the Builder

The builder allows you to generate output files from the data stored in you regenerate project. These files can include (among others):

  • Verilog RTL for the registers
  • Address decoders to allow multiple register blocks to be connected to a single interface
  • UVM register packages for design verification
  • Quasi-static constraints for timing analysis
  • C/C++ #defines
  • IP-XACT files

Rules

The builder uses rules to build output files. Rules are created by the user, and convert a data source to the desired output format.

Data Sources

A data source can be:

  • A register set
  • A block
  • The entire project

The output format determines what type of source is required. For example, the Verilog RTL output for registers required a register set, while the UVM Register Package requires the entire project.

Creating a Rule

Rules are created by use the rule assistant. This is accessed by clicking on the Add button on the right hand side of the window. This brings up the rule assistant, which will guide you thorugh the process of creating the rule. There are up to four steps to creating a rule.

  1. Selecting the output format.
  2. Selecting the source. Note that if the output format requires the entire project, this step is skipped
  3. Selecting any output format specific operation
  4. Selecting the output file

Once a rule has been created, it appears in the builder list

Rules are associated with their data source, not with the project. This allows rules to be imported with the data source when a register set or block is imported into another project.

Building files

Once a rule has been created, building the files is quite simple. Simply clicking the Build button will build all selected files.

The builder attempts to select all out of date files by default
