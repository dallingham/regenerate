Register Set Instances

A register set instance is an instantiation of a regster with in a block. The register set is the definition, and the register set instance is the actual object. In software terms, you can think of the register set as a class and the register set instance as the object. There can be multiple instatiations of the same register set within the block.

When a register set instance is created, it needs to be described to the system with some additional information. This includes:

  • Instance name. Each instance has to have a unique name to distiguish if from the other instances.
  • Offset. Indicates where the instance is located in the address map relative to the other instances. Instance address ranges (the address base + the register set size) may not overlap.
  • Repeat. Indicates if the instance is actually an array of instances or a single instance. If the repeat count is greater than 1, it is considered to be an array. Each instance is a contiguous block in the address map, with next instance immediately beginning where the last instance in the array ended.
  • HDL Path. The HDL path maps to the heirarchical path from the block instance to the register set instance. This is combined with the HDL Path at the top level to create a full path to the register set instace. While optional, this path is used by the static timing and UVM register package exporters. In the HDL path, the index is represented in the path with a %0d value, and must be the same between all instances in the array. It is recommended that a generate statement be used to create instaces of the block in the RTL code.
